## Strategic context

The June 2026 PRD reverses the June 11 "free educational tool" pivot and re-introduces:
- Persona-sequenced MVPs (Maya → Alex → Devon → Priya)
- Freemium subscription (Stripe, ~$7/mo Plus, ~$18/mo Premium)
- B2B API line (targeting MVP 2 timeframe)
- User accounts (implied by watchlist persistence, digest, paywall)

The current codebase was built as a free tool with no auth and no payments. Architecture is otherwise well-aligned: Flask + React, Groq LLM rationale, structured JSON signals, Railway hosting.

## Cross-cutting gaps (block multiple MVPs)

### 1. No user accounts / auth
Blocks: watchlist persistence across sessions (MVP 1), digest subscription (MVP 2), paywalling (MVP 2/3/4), retention measurement (MVP 1 metric).

### 2. No analytics / event tracking
Blocks: measuring every MVP 1 success metric. The PRD's activation rate, time-to-first-idea, and jargon-block rate cannot be tracked without instrumentation.

### 3. Signal schema missing PRD-required fields
The PRD says `interest_tags`, `persona_tags`, and `theme_tags` should be added to the schema now (while cheap) to enable MVP 2 personalization and MVP 4 theme filtering. Current schema doesn't have them.

### 4. `invalidation` / downside inconsistently populated
The UI supports it; the LLM prompt doesn't reliably produce it. This is a prompt-engineering task, not a schema task.

### 5. Watchlist is an orphaned component
`ResearchWatchlist.jsx` exists with localStorage persistence but has no route/tab and no "Save" button on signal cards. Either wire it in or remove it.

## Recommended next steps (priority order)

These are sequenced by PRD priority — MVP 1 must hit its success threshold before MVP 2 begins.

**Immediate (unblock MVP 1 P0):**
1. **Inline glossary** — tappable tooltip on terms like P/E, ETF, market cap in signal cards. Could be a thin `<Tooltip>` component seeded with 10–15 definitions.
2. **Watchlist wire-up** — add "Save" button to signal cards → localStorage watchlist → expose via a 6th tab or a sheet/overlay. (Re-using existing `ResearchWatchlist` component.)
3. **Analytics** — add a lightweight event tracker (e.g., Plausible, PostHog, or a custom `POST /api/event`) so activation rate and time-to-signal can be measured. Without this, MVP 1 success threshold is unmeasurable.

**Short-term (complete MVP 1):**
4. **"What could go wrong" in every signal** — update the Groq prompt to reliably populate `invalidation`; show it on `SignalCard` as well as `SignalCardEnhanced`.
5. **Default tab to signals** — or add a clear "Start Here" CTA on Home that goes directly to "This Week." The PRD's <2 min goal requires fewer taps.
6. **In-app micro-survey** — one-question confidence check after viewing first idea ("Did you understand why this is recommended?") to measure the 70% qualitative target.

**Signal schema (do now, needed later):**
7. **Add `interest_tags`, `theme_tags` to signal JSON** — lightweight string arrays, empty for now. Keeps MVP 4 theming cheap when the time comes.

**MVP 2 (after MVP 1 threshold met):**
8. User accounts (auth) — needed before Stripe, digest, and persistent watchlist.
9. Weekly curated shortlist view (top 3–5 by confidence).
10. Stripe integration.
